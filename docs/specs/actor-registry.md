# Actor registry — Phase 2 spec

*Implements decision 0004. Vault-side first; public cards are a derived,
redacted view that ships only when entries earn it.*

## Purpose

Track individual police personnel and vehicles across independent clips so
that unidentified ("unbadged") actors accumulate a documented record of acts,
and badge concealment itself is evidenced per actor. Primary consumers: the
verification workflow, the legal-handoff package (acts per actor + command
responsibility), and — for corroborated entries only — public actor cards.

## Storage

- `vault/actors/registry.jsonl` — append-only, hash-chained like the intake
  ledger (same `seq`/`prev` discipline, IST timestamps). Each line is an
  event: `actor-created`, `appearance-linked`, `appearance-retracted`,
  `identity-claim`, `status-change`. Current state is derived by replay;
  nothing is ever edited in place.
- Public view: `site/data/actors.json`, generated by tooling from registry
  state — only entries with `publicCard: true`, only public-safe fields.

## Designators

`U-NN` unbadged personnel · `B-NN` badge/name-plate-visible personnel ·
`V-NN` vehicles. Two-digit minimum, assigned sequentially, never reused.
If a U actor is later identified by marker (e.g. a clip surfaces where the
name plate is visible), the designator does NOT change — the marker lands in
`markers` and the history stays legible.

## Record shape (derived state)

```json
{
  "actor": "U-07",
  "kind": "personnel",
  "createdAt": "2026-07-22T10:00:00+05:30",
  "markers": {
    "badgeNumber": null,
    "namePlate": "absent in all appearances",
    "rankInsignia": "unknown",
    "unitMarkings": "e.g. company lettering on helmet/shield if visible",
    "appearance": "free text: kit, build, distinguishing equipment"
  },
  "badgeConcealed": true,
  "appearances": [
    {
      "sha256": "<vault original this was observed in>",
      "clipRange": "00:41-01:03",
      "event": "evt-006",
      "location": "as visible/stated, generalized for public view",
      "act": "neutral description of the act observed",
      "actStatus": "claimed | corroborated",
      "method": "how continuity was established: uniform, kit, position, timeline",
      "linkedBy": "reviewer",
      "linkedAt": "IST timestamp",
      "confidence": "high | medium | low"
    }
  ],
  "identityClaims": [
    {
      "claim": "never published; vault only",
      "basis": "witness statement <submission-ref> | visible marker <sha256>",
      "status": "claimed | corroborated",
      "recordedAt": "IST timestamp"
    }
  ],
  "status": "claimed | corroborated",
  "publicCard": false
}
```

## Rules (enforced by tooling, reviewed by humans)

1. **Linking method is always recorded.** An appearance without a `method`
   string is invalid. Continuity rests on uniform/kit/position/timeline —
   human judgment, documented. No biometric tooling anywhere in the loop,
   including as a reviewer "assist" (decisions 0001, 0004).
2. **`publicCard` requires:** ≥2 appearances, each `actStatus` at least
   claimed, at least one appearance corroborated (independent clip or witness),
   and a second reviewer sign-off recorded in the registry.
3. **Public fields only:** designator, marker observations, badgeConcealed,
   appearances (event link + generalized location + act + status + clip count
   — never the raw vault footage until the redaction pipeline exists).
   `identityClaims` never leave the vault (they route to legal handoff).
4. **Acts are described neutrally** ("strikes seated protester with lathi,
   3 blows visible") — the act description IS the indictment; adjectives
   weaken it.
5. **Retraction is first-class.** A wrong link is corrected by an
   `appearance-retracted` event with a reason — visible in history, because
   honest correction is credibility.

## Badge concealment as evidence

`badgeConcealed: true` + N appearances is itself a public statistic:
"N of M tracked personnel displayed no identification." That number is the
platform's thesis in one line and feeds the command-responsibility argument
(who ordered/tolerated unbadged deployment) — the primary legal target.

## Out of scope

Nicknames/epithets (decision 0004). Face search (0001). Publishing identity
claims. Automated cross-clip person re-identification models — that is
biometrics regardless of marketing name.
